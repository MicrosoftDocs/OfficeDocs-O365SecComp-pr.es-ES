<span data-ttu-id="148ea-104">A partir de la escala de Office 365, sería imposible mantener los datos de los clientes resistentes y protegidos del malware sin una supervisión integrada que sea exhaustiva, lo que alerta de que es inteligente y tiene una solución automática y que es más rápida y fiable.</span><span class="sxs-lookup"><span data-stu-id="148ea-104">Given the scale of Office 365, it would be impossible to keep customer data resilient and safe from malware without built-in monitoring that is comprehensive, alerting that is intelligent, and self-healing that is fast and reliable.</span></span> <span data-ttu-id="148ea-105">La supervisión de un conjunto de servicios en la escala de Office 365 es muy desafiante.</span><span class="sxs-lookup"><span data-stu-id="148ea-105">Monitoring a set of services at the scale of Office 365 is very challenging.</span></span> <span data-ttu-id="148ea-106">Es necesario crear nuevas metodologías y máximas y nuevos conjuntos de tecnología para que funcionen y administren el servicio en un entorno global conectado.</span><span class="sxs-lookup"><span data-stu-id="148ea-106">New mindsets and methodologies needed to be introduced, and whole new sets of technology needed to be created to operate and manage the service in a connected global environment.</span></span> <span data-ttu-id="148ea-107">Hemos alejado el enfoque de supervisión tradicional de recopilación de datos y filtrado para crear alertas de un enfoque basado en análisis de datos; <span data-ttu-id="148ea-115">Office 365 también usa el concepto de *alerta roja*, que reduce todas las señales de supervisión de todos los equipos de nuestros centros de proceso de trabajo a algo manejable por un humano.</span><span class="sxs-lookup"><span data-stu-id="148ea-115">Office 365 also uses the concept of *Red Alert*, which shrinks down all the monitoring signals from all of the machines in our datacenters to something manageable by a human being.</span></span> <span data-ttu-id="148ea-116">El concepto es bastante sencillo: Si ocurre algo en varias señales, debe haber algo en marcha.</span><span class="sxs-lookup"><span data-stu-id="148ea-116">The concept is quite simple: If something is happening across multiple signals, there must be something going on.</span></span> <span data-ttu-id="148ea-117">No se trata de crear confianza en una señal, ya que se trata de tener una fidelidad razonable para cada señal, de modo que pueda obtener una mayor precisión.</span><span class="sxs-lookup"><span data-stu-id="148ea-117">It is not about building confidence in one signal, it is about having reasonable fidelity for each signal so that you get greater accuracy.</span></span> <span data-ttu-id="148ea-118">Este sistema de supervisión es tan potente que no disponemos del personal 24x7 para supervisar nuestros monitores; <span data-ttu-id="148ea-121">Además de las funcionalidades de recuperación automática, como la restauración de una sola página, Exchange Online incluye varias características que adoptan un enfoque para la supervisión y la reparación automática que se centra en preservar la experiencia del usuario final.</span><span class="sxs-lookup"><span data-stu-id="148ea-121">In addition to self-healing capabilities such as single page restore, Exchange Online includes several features that take an approach to monitoring and self-healing which focuses on preserving the end-user experience.</span></span> <span data-ttu-id="148ea-122">Estas características incluyen la *disponibilidad administrada*, que proporciona acciones de recuperación y supervisión integradas, y AutoReseed, que restaura automáticamente la redundancia de la base de datos después de un error en el disco.</span><span class="sxs-lookup"><span data-stu-id="148ea-122">These features include *Managed Availability*, which provides built-in monitoring and recovery actions, and AutoReseed, which automatically restores database redundancy after a disk failure.</span></span>

## <a name="autoreseed"></a><span data-ttu-id="148ea-137">AutoReseed</span><span class="sxs-lookup"><span data-stu-id="148ea-137">AutoReseed</span></span>

<span data-ttu-id="148ea-141">Dado el tamaño de Exchange Online y el hecho que se implementó en él, hay millones de unidades de disco, los errores de la unidad de disco son una ocurrencia regular en Exchange Online.</span><span class="sxs-lookup"><span data-stu-id="148ea-141">Given the size of Exchange Online and the fact that deployed within it are millions of disk drives, disk drive failures are a regular occurrence in Exchange Online.</span></span> <span data-ttu-id="148ea-142">De hecho, más de 100 errores cada día.</span><span class="sxs-lookup"><span data-stu-id="148ea-142">In fact, more than 100 fail every day.</span></span> <span data-ttu-id="148ea-143">Cuando se produce un error en un disco en una implementación empresarial local, un administrador debe reemplazar manualmente el disco con errores y restaurar los datos afectados.</span><span class="sxs-lookup"><span data-stu-id="148ea-143">When a disk fails in an on-premises enterprise deployment, an administrator must manually replace the failed disk and restore the affected data.</span></span> <span data-ttu-id="148ea-144">En una implementación en la nube, el tamaño de Office 365, la necesidad de que los operadores (administradores de nube) reemplacen los discos manualmente no sea práctico ni económicamente viable.</span><span class="sxs-lookup"><span data-stu-id="148ea-144">In a cloud deployment the size of Office 365, having operators (cloud administrators) manually replacing disks is neither practical nor economically feasible.</span></span>
