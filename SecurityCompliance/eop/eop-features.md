# <a name="eop-features"></a>Características de EOP

La tabla siguiente proporciona una lista de las características que están disponibles en el servicio de filtrado de correo hospedado Exchange Online Protection. 
  
> [!TIP]
> La [Guía básica de Office 365 para empresa](https://office.microsoft.com/en-us/products/office-365-roadmap-FX104343353.aspx) es un buen recurso para encontrar información sobre las próximas características nuevas. Para obtener una visión más amplia acerca de las características que están disponibles con los distintos planes de suscripción a EOP, vea [Descripción del servicio Exchange Online Protection](https://go.microsoft.com/fwlink/p/?LinkId=320619).

|:-----|:-----|
|**Característica** <br/> |**Descripción** <br/> |
|**Protección contra correo no deseado** <br/> ||
|Detección de correo no deseado de entrada  <br/> |La protección contra correo no deseado de entrada siempre está habilitada y no se puede deshabilitar. Puede establecer una configuración personalizada mediante directivas de filtrado de conexión y filtrado de contenido.  <br/> Para los clientes de EOP independiente: de forma predeterminada, los filtros de contenido de elevación de privilegios envían los mensajes detectados como correo no deseado a la carpeta de correo no deseado de cada destinatario. Sin embargo, con el fin de ayudar a garantizar que la acción de **mover el mensaje a la carpeta correo no deseado** funcionarán con buzones locales, debe configurar dos reglas de transporte de Exchange en los servidores locales para detectar los encabezados de spam agregados por elevación de privilegios. Para obtener información detallada, vea [Asegúrese de que el correo no deseado se enrute a la carpeta de correo no deseado de cada usuario](../ensure-that-spam-is-routed-to-each-user-s-junk-email-folder.md).           |
|Detección de correo no deseado de salida  <br/> |Protección contra correo no deseado saliente siempre está habilitado si se usa el servicio para el envío de correo electrónico saliente, lo que ayuda a proteger a las organizaciones que usan el servicio y sus destinatarios. Similar a filtrado entrantes, salientes filtrado de spam se compone de filtrado de la conexión y filtrado de contenido. La configuración de filtrado de correo no deseado saliente no es configurable, pero hay correo no deseado saliente configuraciones de directivas que puede usar para configurar notificaciones de administración para los mensajes salientes sospechosos y bloqueados. Para obtener más información, vea [Configurar la directiva de correo no deseado saliente](../configure-the-outbound-spam-policy.md).<br/> |
|Protección contra el reenvío masivo de correo electrónico no deseado de NDR  <br/> |Para obtener más información acerca de contra NDR, consulte el contra NDR configuración en [Opciones de filtrado avanzadas correo no deseado](../advanced-spam-filtering-asf-options.md) , así como [los mensajes de contra y elevación de privilegios](../backscatter-messages-and-eop.md).  <br/> |
|Filtrado de correo masivo  <br/> |Elevación de privilegios ha mejorado métodos de detección para la identificación de los mensajes de correo electrónico de forma masiva. Puede configurar el servicio para marcar mensajes de correo electrónico de forma masiva a través de la interfaz de usuario. También puede crear reglas de transporte a más agresiva correo masivo de filtro mediante la búsqueda de un sello de encabezado del mensaje de correo masivo. Para obtener más información acerca de correo electrónico masivo, consulte [¿Cuál es la diferencia entre correo electrónico no deseado y correo electrónico masivo?](../what-s-the-difference-between-junk-email-and-bulk-email.md) y sus temas asociados.<br/> |
|Listas de bloque de direcciones URL malintencionadas  <br/> |EOP usa varias listas de bloqueo de URL que ayudan a detectar vínculos malintencionados conocidos dentro de los mensajes.  <br/> |
|Protección contra suplantación de identidad  <br/> |EOP incluye 750.000 dominios de remitentes de correo no deseado conocidos.  <br/> |
|**Administración del correo no deseado** <br/> ||

El servicio también es compatible con las direcciones IPv6. Para obtener más información, vea [Configurar la directiva de filtro de conexión](../configure-the-connection-filter-policy.md).<br/> |

Directivas personalizadas siempre tienen prioridad sobre la directiva predeterminada, pero puede cambiar la prioridad (es decir, el orden de ejecución) de las directivas personalizadas. Para obtener más información, vea [configurar sus directivas de filtro de correo no deseado](../configure-your-spam-filter-policies.md).<br/> |
|Capacidad para configurar acciones en mensajes filtrados por contenido  <br/> |Existen varias acciones configurables. Por ejemplo, puede eliminar mensajes filtrados por contenido o enviarlos a la carpeta correo electrónico no deseado o la cuarentena. Para obtener más información, vea [configurar sus directivas de filtro de correo no deseado](../configure-your-spam-filter-policies.md).<br/> |
|Capacidad para configurar opciones avanzadas para el filtrado agresivo de correo no deseado  <br/> |Para obtener más información, vea [configurar sus directivas de filtro de correo no deseado](../configure-your-spam-filter-policies.md) (que es donde puede configurar ellos) y [Opciones de filtrado avanzadas correo no deseado](../advanced-spam-filtering-asf-options.md) (que proporciona detalles específicos sobre lo que hace cada opción).  <br/> |
|Filtrado de correo no deseado internacional  <br/> |Puede configurar EOP para filtrar mensajes escritos en idiomas concretos o enviados desde ciertos países o regiones. Puede configurar hasta 250 diferentes regiones e idiomas diferentes 86. El servicio aplicará la acción configurada como correo no deseado de alta confianza. Para obtener más información, vea [configurar sus directivas de filtro de correo no deseado](../configure-your-spam-filter-policies.md).<br/> |
|Administrar el correo electrónico mediante Outlook o Outlook Web App (OWA)  <br/> |Los administradores y usuarios finales pueden crear listas de remitentes seguros y listas de remitentes bloqueados. Para obtener más información:  <br/> OWA: Vea [Bloquear o permitir (configuración del correo electrónico no deseado)](https://go.microsoft.com/fwlink/p/?LinkId=294862).  <br/> Outlook: Vea [Información general sobre el filtro de correo electrónico no deseado](https://go.microsoft.com/fwlink/p/?LinkId=270065).  <br/> Si usa EOP para ayudar a proteger los buzones locales, asegúrese de usar la sincronización de Active directory para ayudar a garantizar que estas opciones se sincronizan con el servicio. Para obtener más información acerca de cómo configurar la sincronización de Active directory, vea "Usar la sincronización de Active directory para administrar usuarios de correo" en [Administrar usuarios de correo en EOP](manage-mail-users-in-eop.md).           |
|Enviar correo no deseado mediante el complemento Informes de correo electrónico no deseado para Microsoft Office Outlook  <br/> |Puede descargar un complemento para Outlook que le permite enviar mensajes seguros a Microsoft para su análisis. Para obtener más información sobre cómo descargar y utilizar esta herramienta, vea [Habilitar el complemento en el mensaje de informe](https://support.office.com/article/4250c4bc-6102-420b-9e0a-a95064837676).<br/> Si está utilizando Exchange Server 2013 con EOP, también puede secundario en OWA para enviar mensajes seguros, como se describe en [las estafas de suplantación de identidad en Outlook en la web y de correo electrónico no deseado de informe ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).  <br/> |
|Enviar correo no deseado y mensajes que no son correo no deseado mediante un alias de correo electrónico  <br/> |Puede enviar correo no deseado (no deseado) y los mensajes que no sean de correo no deseado (no correo no deseado) a Microsoft a través de correo electrónico. Para obtener más información, vea [enviar spam, no spam y los mensajes de estafas de suplantación de identidad a Microsoft para su análisis](../submit-spam-non-spam-and-phishing-scam-messages-to-microsoft-for-analysis.md).<br/> |
|Enviar correo no deseado y mensajes que no son correo no deseado mediante Notificación de correo electrónico no deseado de OWA  <br/> |Puede enviar no deseado y mensajes seguros a Microsoft a través de correo no deseado OWA. Para obtener más información, vea [las estafas de suplantación de identidad en Outlook en la web y de correo electrónico no deseado de informe ](../report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md).<br/> Esta característica está disponible para clientes de Outlook Web App (OWA) con buzones de correo de Exchange Server 2013 SP1 que estén filtrados por EOP. Los clientes de OWA de Exchange Online también tendrán estas funciones en un futuro próximo.           |

|**Protección antimalware** <br/> ||

Como resultado, el filtrado de malware está habilitado automáticamente para todos los clientes.  <br/> |
|Inspección de malware en el cuerpo del mensaje y los datos adjuntos  <br/> |El servicio inspecciona la carga activa en el cuerpo del mensaje y en todos los archivos adjuntos a mensajes en busca de malware.  <br/> |
|Notificaciones de alerta de malware predeterminadas o personalizadas  <br/> |Tiene la opción para enviar un mensaje de correo electrónico de notificación a los remitentes o los administradores cuando un mensaje se detecta como malware y no se entrega. Para obtener más información, vea [Configure anti-malware policies](../configure-anti-malware-policies.md).<br/> |

Directivas personalizadas siempre tienen prioridad sobre la directiva predeterminada, pero puede cambiar la prioridad (es decir, el orden de ejecución) de las directivas personalizadas. |**Reglas de transporte** <br/> ||
|Filtrado y acciones basados en directivas  <br/> |Directivas personalizadas se basan en las reglas de transporte de Exchange. Puede filtrar por dominio, palabra clave, nombre de archivo, tipo de archivo, línea de asunto, cuerpo del mensaje, remitente, destinatario, encabezado y dirección IP. Para obtener más información, vea [Mail flow rules (reglas de transporte) en Exchange Online Protection](mail-flow-rules-transport-rules-0.md).<br/> |
|Filtrar por patrones de texto  <br/> |Las reglas de transporte pueden usar una matriz o expresiones regulares para buscar texto. También puede usar una cadena o una matriz de cadenas para buscar por varias propiedades de mensaje, como dirección, asunto, cuerpo o nombres de datos adjuntos. Para obtener más información, vea [Transport Rule Conditions (Predicates)](http://technet.microsoft.com/library/04edeaba-afd4-4207-b2cb-51bcc44e483c.aspx).<br/> |
|Diccionarios personalizados  <br/> |Las reglas de transporte pueden incluir largas listas de texto y palabras clave, con lo que ofrecen la misma función que un diccionario personalizado.  <br/> |
|Reglas de directiva por dominio  <br/> |El ámbito de una regla de transporte puede personalizarse para buscar por nombre de remitente o de dominio del destinatario, intervalos de direcciones IP, palabras clave o patrones de dirección, pertenencia a grupos y otras condiciones. Para obtener más información, vea [Transport Rule Conditions (Predicates)](http://technet.microsoft.com/library/04edeaba-afd4-4207-b2cb-51bcc44e483c.aspx).<br/> |
|Examen de archivos adjuntos  <br/> |Se pueden crear reglas para examinar el nombre de archivo, la extensión y el contenido de los datos adjuntos.  <br/> |
|Enviar notificaciones sobre las reglas de directiva al remitente  <br/> |Puede rechazar los mensajes y enviar un informe de no entrega (NDR) al remitente mediante las acciones **Rechazar el mensaje con la explicación** o **Rechazar el mensaje con el código de estado mejorado**. Para obtener más información, vea [Transport rule actions](http://technet.microsoft.com/library/f8621ecb-a177-4025-9011-a6569999746a.aspx).<br/> |
|Enviar mensajes a direcciones fijas (como redirigir o copiar un mensaje a una dirección específica)  <br/> |Las reglas de transporte pueden redirigir, agregar destinatarios a copia o copia oculta, simplemente agregar destinatarios y otras opciones. Para obtener más información, vea [Transport rule actions](http://technet.microsoft.com/library/f8621ecb-a177-4025-9011-a6569999746a.aspx).<br/> |
|Capacidad de ajustar fácilmente la prioridad de la regla en varias reglas  <br/> |Use el Centro de administración de Exchange para cambiar el orden en que se procesan las reglas. Para obtener más información, vea [Mail flow rules (reglas de transporte) en Exchange Online Protection](mail-flow-rules-transport-rules-0.md).<br/> |
|Cambiar el nivel de confianza de correo no deseado de un mensaje por regla.  <br/> |Puede inspeccionar un mensaje en tránsito y asignar un nivel de confianza de correo no deseado basándose en los criterios que elija. Para obtener más información, vea [Usar reglas de flujo de correo para establecer el nivel de confianza de correo no deseado (SCL) en los mensajes](../use-mail-flow-rules-to-set-the-spam-confidence-level-scl-in-messages.md).<br/> |
